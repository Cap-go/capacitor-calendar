# AGENTS.md

Guidance for agents and contributors working on `@capgo/capacitor-calendar`.

## Commands

Use Bun for local development commands.

```bash
bun install
bun run build
bun run verify
bun run lint
bun run fmt
```

Individual platform checks:

```bash
bun run verify:ios
bun run verify:android
bun run verify:web
```

## Project Structure

- `src/definitions.ts` and `src/sub-definitions/` define the public TypeScript API.
- `src/schemas/` contains exported option/result/enumeration types.
- `ios/Sources/CapacitorCalendarPlugin/` contains the Swift implementation.
- `android/src/main/kotlin/app/capgo/calendar/` contains the Kotlin implementation.
- `dist/` is generated by `bun run build`; do not edit it manually.

## Native Wiring

The runtime plugin name must stay `CapacitorCalendar` across:

- `registerPlugin('CapacitorCalendar')` in `src/index.ts`
- `@CapacitorPlugin(name = "CapacitorCalendar")` on Android
- `PluginConfig.jsName = "CapacitorCalendar"` on iOS

Run `bun run check:wiring` after changing plugin names, package IDs, podspecs, or SwiftPM configuration.

## Release Notes

- The plugin targets Capacitor 8 and starts at version `8.0.0`.
- iOS must continue to support both CocoaPods (`CapgoCapacitorCalendar.podspec`) and Swift Package Manager (`Package.swift`).
- API docs in `README.md` are generated from TypeScript comments; update source JSDoc and run `bun run docgen`.
- Keep the upstream MIT attribution in `THIRD_PARTY_NOTICES.md` for code derived from `ebarooni/capacitor-calendar`.

## Timeout Policy

- Keep CI, script, and runtime timeouts at 10 minutes or less. Use `timeout-minutes: 10` or lower in GitHub Actions and cap timeout values at `600000` ms, `600` seconds, or `10m` unless explicitly requested.
